## 1. 研究领域

当前主要关注两个方向：

1. `Wan2.1` 视频扩散模型的少步蒸馏与加速，重点是基于 `FastGen` 的 DMD2 / CD / ECT / MeanFlow / f-distill 等方法复现与对比。
2. 面向视频生成/编辑的 `LoRA` 低成本适配与潜在蒸馏结合方案，重点关注视频中的时序建模、运动/外观解耦和显存效率。

## 2. 领域核心问题

当前核心问题有三类：

1. **蒸馏 pipeline 的工程可运行性与可复现性。**
   FastGen 的多方法训练入口、FSDP 配置、checkpoint 组织、推理验证路径需要先完全跑顺，否则后续 PVTT 任务适配会缺少稳定底座。

2. **数据形态与实验效率之间的矛盾。**
   OpenVid-1M 这类 raw 视频数据更贴近通用训练输入，但训练成本高、链路更长；而 synthetic latent 数据显存更友好、训练更快，但由于已经过预处理，和最终真实业务数据链路不完全一致，当前还不能直接作为 Phase 1 的标准输入。

3. **视频 LoRA 是否能作为蒸馏前的低成本探索工具。**
   与图像不同，视频 LoRA 不仅要学习 appearance，还要处理 motion 和 temporal consistency。直接把图像 LoRA 思路搬到视频上，容易出现外观与运动耦合、时序不稳、泛化差的问题。

我当前要解决的具体问题是：
- 先把服务器上的 FastGen 训练/推理 pipeline 梳理清楚；
- 明确 synthetic latent 数据在 Phase 0 中的可用范围；
- 评估视频 LoRA 是否适合作为后续 PVTT 蒸馏前的小规模先导实验。

## 3. 技术方案

本阶段采用以下技术路线：

1. **FastGen 复现主线**
   - 基座模型：`Wan2.1-T2V-1.3B`
   - 训练框架：`FastGen`
   - 方法对比：`DMD2 / CD / ECT / MeanFlow / f-distill`
   - 训练方式：单卡验证 + 多卡 FSDP 训练

2. **数据策略**
   - 早期尝试过 `OpenVid-1M -> WebDataset(mp4+txt)` 路线；
   - 当前更可用的是服务器上已经整理好的 **synthetic latent dataset**，用于快速训练测试与显存验证；
   - 该数据已经过 latent / text embedding 预处理，适合高效实验，但暂时不等同于最终业务原始视频输入。

3. **视频 LoRA 调研与后续实验判断**
   - 根据 `AnimateDiff`，视频 LoRA 适合作为已有视频/图像生成底座上的轻量 motion adaptation；其 `MotionLoRA` 思路强调在已有 motion module 上做低成本增量适配，而不是直接替代完整视频训练流程。
   - 根据 `MotionDirector`，视频 LoRA 的关键不是单纯“插 LoRA”，而是 **外观与运动解耦**；其 dual-path LoRA 和 appearance-debiased temporal loss 说明视频任务里 temporal branch 需要单独建模。
   - 根据 Hugging Face `Diffusers / finetrainers` 的当前生态，视频 LoRA 已经可以较低成本训练 `CogVideoX / HunyuanVideo / Wan`，适合做小规模验证、风格/动作适配和环境熟悉，但**并不是现成的少步蒸馏替代方案**。

## 5. 结论与发现

本周得到的主要结论如下：

1. **Phase 0 的主线工作基本已经跑通。**
   服务器上的真实进展比本地 repo 记录更完整，核心训练和推理结果已经积累下来，当前问题更多是整理、归档和形成稳定入口，而不是从零开始。

2. **synthetic latent 数据非常适合做 Phase 0 的快速验证。**
   它显著降低了训练显存压力，使多方法比较和参数试验更容易推进。这对熟悉 FastGen、验证配置、检查 checkpoint / inference 流程很有帮助。

3. **但 synthetic latent 数据不能直接替代后续真实任务数据。**
   这是当前最重要的边界条件。它更像是“快速实验介质”，而不是最终 PVTT 训练输入。后续如果要做面向业务的蒸馏或视频编辑适配，仍需要重新面对 raw 视频链路。

4. **视频 LoRA 值得继续研究，但它更适合作为“轻量适配工具”，不是现成蒸馏终解。**
   我当前的判断是：
   - 它适合快速验证特定 motion / style / domain adaptation；
   - 适合作为进入视频训练环境的低成本入口；
   - 也可能适合作为后续蒸馏中的 teacher/student 适配辅助手段；
   - 但“直接用 LoRA 解决 50->4 步蒸馏”这件事目前没有成熟现成路径，至少我现在看到的主流视频 LoRA 工作核心仍是适配，不是 step distillation。

5. **视频 LoRA 的关键难点不是参数量，而是时序建模。**
   从调研看，真正有效的方法都在强调 temporal module、motion branch 或 motion/appearance 解耦。这说明后续如果做视频 LoRA 测试，不能完全按图像 LoRA 经验来设计实验。

## 附录

### 视频 LoRA 调研摘要

- `AnimateDiff` 提出可插拔 motion module，并进一步提出 `MotionLoRA`，说明视频 LoRA 可以作为已有生成底座上的低成本 motion adaptation 工具。  
  Source: https://arxiv.org/abs/2307.04725

- `MotionDirector` 指出视频 motion customization 中容易出现 appearance 和 motion 耦合，因此采用 dual-path LoRA 与 appearance-debiased temporal loss。  
  Source: https://arxiv.org/abs/2310.08465

- Hugging Face `Diffusers` 的 `CogVideoX` 训练文档给出了较具体的 LoRA 实践建议：长描述 caption、更统一的视频预处理、以及小数据场景下的 step / rank 经验值。  
  Source: https://huggingface.co/docs/diffusers/main/en/training/cogvideox

- Hugging Face `finetrainers` 已支持 `CogVideoX / HunyuanVideo / Wan` 的视频 LoRA / full finetuning，并强调 memory-efficient precomputation、single-GPU training 和可复现实验脚本。  
  Source: https://github.com/huggingface/finetrainers

- `musubi-tuner` 已支持 `Wan2.1/2.2` 视频 LoRA 训练，说明社区工具链已经在向视频 LoRA 快速迭代，但该项目是非官方实现，适合参考，不宜直接当作唯一基线。  
  Source: https://github.com/kohya-ss/musubi-tuner

### 当前判断

基于本周实验和上述资料，我的当前判断是：

- **短期**：视频 LoRA 适合作为低成本先导实验，帮助熟悉环境、数据格式与时序适配问题。
- **中期**：真正的少步蒸馏主线仍应围绕 FastGen / DMD2 / consistency / progressive distillation 展开。
- **后续值得测试的方向**：
  1. 是否可以把 LoRA 用作 student 侧轻量适配层；
  2. 是否可以把外观 LoRA 与运动 LoRA 分离；
  3. 是否能先用 LoRA 做 domain adaptation，再进入蒸馏训练。
